Calendars

w-calendar is a responsive scheduling surface for month, week, day, four-day, custom, and category views. It accepts event and category arrays, renders all-day and timed events, and exposes a vanilla JavaScript API with native event names.

Month

The month view displays all-day and timed events across a complete calendar grid. Select a date, use the header controls, or call next(), prev(), and today().

The category view compares schedules side by side. Categories accept a JSON string or a JavaScript array; events map through event-category.

Custom Ranges

custom-weekly renders week rows between explicit dates; custom-daily renders interval columns. max-days caps open-ended daily ranges.

Event Mapping and Overflow

Map alternative event fields with event-name, event-start, event-end, event-timed, and event-color. Month cells show three events before a configurable “more” action.

Custom Content

Vue slots become framework-neutral rendering callbacks in DuVay. Use dayContent, dayHeaderContent, dayBodyContent, intervalContent, intervalHeaderContent, eventContent, and categoryContent to return application-owned markup.

Native Drag and Drop

event-draggable makes event controls draggable. Dropping a timed event onto an interval preserves its duration, updates the source event, and emits change with reason: "event-drop". Native drag events continue to bubble for application-specific behavior.

Events

All events are native CustomEvents (bubbles: true, composed: true) carrying a detail object.

EventdetailFires when
change{ reason, value, start, end }The visible range changes. reason is select, move, today, check, or event-drop.
input{ value, ... }A date is selected (model value updates).
click{ kind, date, event?, events?, hidden? }A day, interval, event, or "more" affordance is clicked. kind is day, time, timeCategory, event, or more.
moved{ reason, value, start, end }Navigation moves the view via next(), prev(), or today().
