Tabs

Organize content into switchable panels. DuVay tabs support alignment, equal-width layouts, vertical orientation, stacked icons, overflow pagination, theming, and an animated selection indicator that glides between tabs.

The indicator is progressively enhanced: hand-authored markup shows a static underline, while the <w-tabs> web component renders an animated slider (and falls back to the underline when JavaScript is absent). Pair tabs with <w-tabs-window> for the matching content panes.

Basic tabs

Mark the selected tab with the active class, or set value on <w-tabs> to the active tab's value. The component emits a w-change event carrying the new value in event.detail when the selection changes.

Tabs with content

<w-tabs-window> and <w-tabs-window-item> are Vuetify-named aliases for <w-window> / <w-window-item>. Listen for w-change on the tabs and set the window's value to the active index to keep them in sync.

Use align-tabs (start · center · end) to position the strip. With CSS classes, add w-tabs--align-center or w-tabs--align-end.

Fixed and grow

fixed-tabs gives every tab an equal share of the width, capped at --w-tab-max-width (18rem) and centered. grow spreads the tabs to fill the whole strip with no cap.

Density

Use density (comfortable or compact) to tighten the tab height. With CSS classes, add w-tabs--comfortable or w-tabs--compact.

Color and slider color

color sets the active tab's text and the default indicator color; slider-color overrides just the indicator; bg-color tints the strip. Values are DuVay palette tokens (e.g. primary, success, tertiary). With CSS classes, set the --w-tabs-color, --w-tabs-slider-color, and --w-tabs-bg custom properties.

Hide slider

Add hide-slider (or w-tabs--no-slider) to remove the indicator entirely — the active tab is shown by color alone.

Overflow arrows

When a strip is too narrow for its tabs it scrolls. Add show-arrows to the web component for pagination buttons (customizable with prev-icon / next-icon), and center-active to keep the selected tab scrolled to center. Arrows are a web-component enhancement; with plain CSS the strip simply scrolls.

Give <w-tab> an href to render a real anchor that participates in navigation. With CSS classes, use an <a class="w-tab">.

Disabled

Pills

Set variant="pills" (or add w-tabs-pills) for a rounded, background-highlighted style. The sliding indicator is suppressed in favor of the pill background.

Keyboard

Tabs are a role="tablist". Arrow keys move between tabs (Left/Right when horizontal, Up/Down when vertical) and skip disabled tabs, wrapping at the ends. Home and End jump to the first and last tab. Every tab has a visible :focus-visible ring and a 44px touch target by default.

API reference

Components: <w-tabs>, <w-tab>, <w-tabs-window>, <w-tabs-window-item>.

<w-tabs> attributes: value, variant="pills", align-tabs, fixed-tabs, grow, direction, stacked, center-active, hide-slider, slider-color, color, bg-color, density, disabled, show-arrows, prev-icon, next-icon. Emits w-change with the new value in event.detail.

<w-tab> attributes: value, active, disabled, stacked, href.
