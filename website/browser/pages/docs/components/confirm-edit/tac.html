Confirm Edit

w-confirm-edit keeps changes on a temporary copy of a value until the user commits (OK) or reverts (Cancel). OK stays disabled until the edit differs from the original. Edit with the built-in field, or wrap any control that exposes a value — a text field, textarea, select, or a native input.

Basic

With no slotted editor, a built-in input is rendered from label. Type to enable OK.

Custom action labels

Override the button text with cancel-text and ok-text.

Custom editor

Drop any control into the default slot — it’s bound to value, and its changes drive the pristine state.

Disabled

Add disabled to lock the editor and both actions.

Custom actions

Provide your own buttons in the actions slot and mark them data-save / data-cancel; the component wires the rest. With hide-actions you can omit the buttons entirely and call save() / cancel() yourself.
