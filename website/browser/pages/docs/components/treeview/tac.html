Treeview

Treeviews display hierarchical data with expandable branches. Pass nested data through items (a JSON array of { title, value, children } nodes) or the shorthand A>B>C path syntax. Rows can be activatable (single highlight) or selectable (cascading checkboxes).

Basic

open-all expands every branch; otherwise branches start collapsed and open on the toggle. Bind initial open branches with opened.

Activatable

With activatable, clicking a row highlights it and the treeview reports the value through a change event (detail.name === 'activated'). Add multiple-active to keep more than one row active.

Selectable & selection types

With selectable, each row gets a checkbox and the treeview reports a selected array. select-strategy controls cascade behaviour:

  • leaf (default) — only leaf values are stored; a branch checkbox is checked when all of its leaves are, and indeterminate when some are.
  • classic — branches are stored too; selecting a branch selects it and every descendant.
  • independent — every checkbox toggles on its own with no cascade.

Open on click

open-on-click expands a branch when its row (not just the toggle) is clicked.

Density, rounded & hoverable

density="compact" tightens row height, rounded pills the rows, and hoverable highlights rows on hover. color sets the active/selected accent.
