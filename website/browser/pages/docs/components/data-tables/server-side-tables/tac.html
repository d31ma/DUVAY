Server-side Tables

w-data-table-server shares all of data table’s chrome but does not filter, sort, or paginate locally. Instead items is the current page already prepared by your backend, and items-length is the total row count that drives the footer. When the user changes the page, page size, or sort, it emits update:options so you can fetch the matching slice and update items / items-length.

Example

The table holds only the current page of items; items-length tells the footer there are 50 rows in total, so the pagination controls reflect the full set. Changing the page, size, or sort emits update:options for your fetch handler (wired below).

Wiring it up

In application code, listen for update:options, fetch, then set the new items and items-length:

const table = document.querySelector('w-data-table-server');

table.addEventListener('update:options', async (e) => {
  const { page, itemsPerPage, sortBy, sortDesc } = e.detail;
  table.setAttribute('loading', '');
  const res = await fetch(`/api/desserts?page=${page}&size=${itemsPerPage}&sort=${sortBy}&desc=${sortDesc}`);
  const { rows, total } = await res.json();
  table.items = rows;                       // current page only
  table.setAttribute('items-length', total); // total across all pages
  table.removeAttribute('loading');
});
