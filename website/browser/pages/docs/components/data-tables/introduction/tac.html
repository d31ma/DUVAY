Data Tables

w-data-table renders a semantic <table> and owns the client-side pipeline — filter (search) → sort → paginate. It supports rich or simple headers, single & multi-column sorting, row selection, expandable detail rows, a paginated footer with a per-page select, and loading / empty states.

  • Basics — sorting, search, and pagination.
  • Data and Display — rich headers, selection, expansion, density, and states.
  • Server-side tables — remote data with update:options.
  • Virtual tables — windowed rendering for very long lists.

Basic

Pass headers as a simple comma list (the label doubles as the key) and items as a|b|c; … rows or a JSON array.
