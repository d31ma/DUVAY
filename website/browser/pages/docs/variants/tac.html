Variants

Variants control the visual weight and interaction style of a component. Most DuVay controls that accept a variant attribute share the same vocabulary: filled, outlined, text, tonal, and plain. Buttons also expose icon for icon-only actions.

The color attribute composes with variants. The same semantic tokens work on filled, outlined, and tonal buttons.

Form-control variants

Inputs, selects, text fields, and textareas support the same core variants. The default is usually outlined; set filled for a solid background field.

Choosing a variant

  • Use filled for the single most important action on a surface.
  • Use outlined for secondary actions or dismissal buttons.
  • Use text or plain for low-priority actions inside dense lists or toolbars.
  • Use tonal when you need more emphasis than text but less than filled.
  • Use icon sparingly, and always pair it with an aria-label.
