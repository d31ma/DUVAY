v-model

DuVay web components support two-way binding through a value or model-value attribute and standard events. Use an explicit attribute + event pair to recreate Vue's v-model, React controlled components, or plain DOM scripting without a framework-specific adapter.

Usage

Binding models

The model-value attribute is the Vuetify/Vue v-model alias. When a component's state changes, it dispatches an update:model-value event with the new value. Form controls also accept value and emit input and change events.

<!-- Vue --> <template> <w-alert :model-value="show" @update:model-value="show = $event.detail"> Saved successfully </w-alert> <w-btn @click="show = !show">Toggle</w-btn> </template> <script setup> import { shallowRef } from 'vue' const show = shallowRef(true) </script>

The same binding in vanilla JavaScript:

<w-alert id="alert" model-value="true">Saved successfully</w-alert> <button id="toggle">Toggle</button> <script> const alert = document.getElementById('alert') const btn = document.getElementById('toggle') btn.addEventListener('click', () => { const open = alert.getAttribute('model-value') === 'true' alert.setAttribute('model-value', String(!open)) }) alert.addEventListener('update:model-value', (e) => { console.log('open:', e.detail) }) </script>

Controlled and uncontrolled

If you only set model-value once, the component starts uncontrolled and manages its own state. To control it externally, always reflect the new value back to the attribute in the event listener.

// Uncontrolled: initial value only <w-alert model-value="true">...</w-alert> // Controlled: state lives in the parent <w-rating id="rating" model-value="3"></w-rating> <script> document.getElementById('rating') .addEventListener('update:model-value', (e) => e.target.setAttribute('model-value', e.detail)) </script>

Binding specific properties

Some components expose additional bindable properties such as active, open, or selected. Use them as regular attributes and listen for update:active, update:open, or update:selected. DuVay also fires colon-free aliases — update-active, update-open, update-selected — so Angular and Svelte templates can bind them.

<w-item-group :selected="selected" @update:selected="selected = $event.detail" > <w-item value="a" title="A"></w-item> <w-item value="b" title="B"></w-item> </w-item-group>
