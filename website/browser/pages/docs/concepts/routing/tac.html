Routing

DuVay has no opinion about your router. Because components render in the Light DOM with plain HTML classes, they work with any routing library — or with no router at all.

<!-- Any router can render DuVay markup --> <a href="/dashboard" class="w-btn w-btn-filled">Dashboard</a> <div class="w-page w-fade-in">…</div>

Client-side navigation

When a client-side router swaps content, DuVay web components in the new content automatically upgrade because they are standard custom elements. No framework adapter is needed.

Deep-linking state

Store active tabs, open dialogs, and selected options in the URL when your application needs shareable state. DuVay components emit events so the URL can stay the source of truth.
