File inputs

w-file-input is a labelled file picker. Use accept to restrict file types, multiple to allow several files, and disabled to lock it.

API

  • label — button or field label.
  • accept — comma-separated accepted MIME types or extensions.
  • multiple — allow selecting several files.
  • disabled, readonly — lock the input; readonly also prevents the file dialog.
  • chips / small-chips — show selected files as chips.
  • counter — show the file count and total size.
  • show-size — show each file size inside its chip.
  • clearable — show a clear-all button when files are selected.
  • placeholder — text shown when no files are selected.
  • hint — helper text below the field.
  • error — error message; also applies error styling.
  • truncate-length — max characters for a file name before truncation (default 22).
  • density — compact or comfortable.
  • name — native form field name forwarded to the file input.

Events: input on every selection and change when files change or clear.
