Download

DuVay ships in two flavours that share one design system: CSS classes (style with utility and component classes) and optional web components (drop-in custom elements). Both are zero-dependency. Use the GitHub Pages URLs, download the files, or pin to a specific CalVer release.

Quick start (hosted)

Paste into your <head>. The minified builds are published to GitHub Pages with every release.

CSS classes

<link rel="stylesheet" href="https://d31ma.github.io/DUVAY/latest/duvay.min.css"> <script src="https://d31ma.github.io/DUVAY/latest/duvay.min.js" defer></script>

Selective CSS

@import "duvay-css/core.css"; @import "duvay-css/components/buttons.css"; @import "duvay-css/components/tooltips.css";

Web components

<link rel="stylesheet" href="https://d31ma.github.io/DUVAY/latest/duvay.min.css"> <script type="module" src="https://d31ma.github.io/DUVAY/latest/duvay-wc.min.js"></script> <w-btn variant="filled">Click me</w-btn>

Pin to a CalVer release

Every release is kept at a permanent URL. Swap VERSION for the CalVer you want (for example 26.24.12).

<link rel="stylesheet" href="https://d31ma.github.io/DUVAY/versions/VERSION/duvay.min.css"> <script src="https://d31ma.github.io/DUVAY/versions/VERSION/duvay.min.js" defer></script> <!-- optional web components --> <script type="module" src="https://d31ma.github.io/DUVAY/versions/VERSION/duvay-wc.min.js"></script>

Versioned URLs never change, so you can upgrade on your own schedule by updating the version segment.

Direct download

Self-hosted builds — full (readable) and minified (production). Right-click → Save, or click to download.

The behaviour layer (duvay.js) wires CSS-class interactions — dialogs, dropdowns, toasts, theme cycling — through data-w-* attributes. The web-component bundle is self-contained and does not require it.

Versions

Releases follow CalVer (YY.WW.DD). Replace VERSION with the CalVer release you want; every release is permanently available at https://d31ma.github.io/DUVAY/versions/VERSION/. Always test an upgrade against your app before moving the version pin.
