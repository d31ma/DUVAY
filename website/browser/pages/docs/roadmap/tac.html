Roadmap

DuVay tracks Vuetify and Material Design patterns where they help developer ergonomics, while keeping DuVay lightweight, token-driven, and visually distinct.

Current focus

  • Complete docs coverage for the Vuetify component groups.
  • Polish components for inputs, navigation, overlays, workflow surfaces, tables, and typography.
  • Keep CSS-class and web-component examples aligned on every component page.

Next passes

  • Deepen API references for common attributes, events, slots, and behavior hooks.
  • Promote stable Labs work into the main component docs.
  • Add more route-level visual checks across desktop and mobile docs pages.

Scope guardrails

DuVay should feel familiar to Vuetify users, but it is not a Vue runtime. The framework stays zero-dependency, HTML-first, and portable across static sites and app shells.
