Tooltip directive

The DuVay equivalent of Vuetify's v-tooltip directive — a shorthand way of adding a tooltip to any element. Where Vuetify wraps the <v-tooltip> component, DuVay implements the directive form purely in CSS: add a class and an attribute, and the tooltip renders on hover and keyboard focus with no JavaScript.

Interactive demo

Hover or focus the buttons below to reveal their tooltips. This directive is pure CSS — it works with just duvay.css, no JavaScript required.

Usage

Add the w-tooltip class and a w-tooltip attribute to any element. The text in w-tooltip is rendered through a CSS ::after pseudo-element and revealed on :hover, :focus-visible, and :focus-within.

<button class="w-btn w-btn-text w-tooltip" w-tooltip="Save changes"> Save </button>

Position

In Vuetify the location is passed as a directive argument. In DuVay you set placement with the w-tooltip-position attribute (or the equivalent class form). The default placement is top.

Attribute valueClass formPlacement
w-tooltip-position="top"defaultAbove the element (default).
w-tooltip-position="bottom"w-tooltip--bottomBelow the element.
w-tooltip-position="left"w-tooltip--left (alias w-tooltip--start)To the left of the element.
w-tooltip-position="right"w-tooltip--right (alias w-tooltip--end)To the right of the element.
<button class="w-btn w-tooltip" w-tooltip="Below" w-tooltip-position="bottom">Bottom</button> <button class="w-btn w-tooltip" w-tooltip="Left" w-tooltip-position="left">Left</button> <button class="w-btn w-tooltip w-tooltip--right" w-tooltip="Right">Right (class form)</button>

Tooltip text

Unlike Vuetify — where the directive falls back to the target element's textContent — the DuVay tooltip text always comes from the w-tooltip attribute. This keeps the visible label and the tooltip independent, so a button reading “Save” can show an explanatory “Save changes” on hover.

<span class="w-tooltip" w-tooltip="Copied to clipboard" tabindex="0">Copy</span>

CSS-only — no JavaScript, no event

The directive is implemented entirely with CSS. There is no behavior-layer JavaScript and no event — the tooltip is a ::after pseudo-element that fades in on :hover, :focus-visible, and :focus-within. Because it relies on focus, give non-focusable elements a tabindex="0" so keyboard users can reveal the tooltip too. The text is a single line (white-space: nowrap); keep it short.

When to use the component instead

The CSS-only directive is ideal for short, static hints. For richer needs — interactive content that stays open while hovered, cursor-following tooltips, longer wrapping text, or full location control — reach for the <w-tooltip> web component documented on the Tooltips page.
