Ripple

The DuVay equivalent of Vuetify's v-ripple directive. Add w-ripple to any block-level element and the behavior layer paints a water-ripple effect on press, giving visual feedback for user actions. It can be applied to any interactive surface — buttons, cards, list items — and requires duvay-directives.js to be loaded. The ripple is purely presentational: it dispatches no event.

Usage

Enable basic ripple functionality by adding the w-ripple attribute to a component or HTML element. On pointer down (and on Enter / Space keydown) the behavior layer adds the class w-ripple-host to the element and appends a .w-ripple-ink span at the pointer position. The span removes itself when its animation ends.

<button class="w-btn w-btn-filled" w-ripple>Press me</button>

API

Ripple on web components

Interactive web components accept an opt-in ripple attribute (matching Vuetify's ripple prop) so you do not need the directive on every element. It is off by default; add the attribute to enable it.

Web component + ripple

Supported on <w-btn>, <w-list-item>, <w-chip>, and <w-tab>.

Examples

Options

By default the ripple originates from the pointer position. Add w-ripple-center so it always originates from the center of the target, mirroring Vuetify's .center modifier.

<button class="w-btn w-btn-filled" w-ripple w-ripple-center>Centered</button>

Custom color

Use w-ripple-color to adjust the color of the ink. The value maps to a design token, so the tint stays theme-aware across light and dark.

<button class="w-btn w-btn-outline" w-ripple w-ripple-color="primary">Primary ink</button> <button class="w-btn w-btn-outline" w-ripple w-ripple-color="secondary">Secondary ink</button> <button class="w-btn w-btn-outline" w-ripple w-ripple-color="surface">Surface ink</button>

Disabling the ripple

Where a component opts into the ripple, set w-ripple="false" to turn it off for a single element without removing the styling.

<button class="w-btn w-btn-filled" w-ripple="false">No ripple</button>

Ripple on a custom surface

The directive is not limited to buttons. Apply it to any block-level element — a card or a list item — to add press feedback.

<article class="w-card" w-ripple> <h3 class="w-card-title">Tap me</h3> <p>The whole card ripples on press.</p> </article> <ul class="w-list"> <li class="w-list-item" w-ripple>Inbox</li> <li class="w-list-item" w-ripple w-ripple-center>Drafts</li> </ul>

Accessibility

The ripple is decorative and adds no semantics, so it never interferes with assistive technology — keep the element's own role, label, and keyboard behavior intact. Because the effect also fires on Enter / Space keydown, keyboard users get the same feedback as pointer users.

Reduced motion

The ripple animation honors prefers-reduced-motion: reduce. Users who request less motion get the press registered without the animated ink, so the feedback stays instant and non-animated.
