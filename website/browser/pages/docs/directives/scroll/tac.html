Scroll

The DuVay equivalent of Vuetify's v-scroll directive. Add w-scroll to any element and DuVay invokes a bubbling w-scroll event when the screen — or a chosen element — is scrolled. By default it watches the window; options let you switch the listener to the element itself or to another target. Requires duvay-directives.js.

Usage

The simplest case listens to the window. Add w-scroll to an element, then attach a w-scroll listener. Each event carries detail.scrollTop, detail.scrollLeft, and detail.target. The underlying scroll listener is passive.

<div w-scroll id="watcher"></div>

<script>
  document.getElementById('watcher')
    .addEventListener('w-scroll', (e) => {
      console.log(e.detail.scrollTop);
    });
</script>

Options

Two attributes change what the directive observes. w-scroll-self mirrors Vue's .self modifier; w-scroll-target mirrors Vuetify's :target argument. When both are present, -self takes precedence.

AttributeVuetify equivalentDescription
w-scrollv-scrollListen on window (the default).
w-scroll-selfv-scroll.selfListen on the bound element itself rather than the window.
w-scroll-target="#sel"v-scroll:targetListen on another element matched by the CSS selector.

Self

By default w-scroll watches the window. Add w-scroll-self to instead observe scrolling within the bound element — useful for a scrollable card or panel whose own content moves.

<div class="w-card" w-scroll w-scroll-self
     style="height: 200px; overflow: auto;" id="panel">
  <!-- long content -->
</div>

<script>
  let count = 0;
  document.getElementById('panel')
    .addEventListener('w-scroll', () => {
      count += 1;
      console.log('scrolled', count, 'times');
    });
</script>

Target

For finer control, point the listener at a different element with w-scroll-target. The value is any CSS selector; the w-scroll event still fires on the bound element, while the scroll position reported in e.detail comes from the target.

<div id="scroll-area" style="height: 300px; overflow: auto;">
  <!-- long content -->
</div>

<div w-scroll w-scroll-target="#scroll-area" id="reporter"></div>

<script>
  document.getElementById('reporter')
    .addEventListener('w-scroll', (e) => {
      console.log('area scrolled to', e.detail.scrollTop);
    });
</script>

Events

The directive dispatches a bubbling CustomEvent named w-scroll. Read scroll position from the detail object.

PropertyDescription
detail.scrollTopVertical scroll offset of the watched element or window.
detail.scrollLeftHorizontal scroll offset of the watched element or window.
detail.targetThe element (or window) the scroll was observed on.
 
el.addEventListener('w-scroll', (e) => {
  const { scrollTop, scrollLeft, target } = e.detail;
  console.log(scrollTop, scrollLeft, target);
});

Examples

Back-to-top button

Reveal a button once the window scrolls past a threshold, then scroll smoothly to the top when clicked.

<button class="w-btn" id="to-top" w-scroll hidden>
  Back to top
</button>

<script>
  const btn = document.getElementById('to-top');
  btn.addEventListener('w-scroll', (e) => {
    btn.hidden = e.detail.scrollTop < 400;
  });
  btn.addEventListener('click', () => {
    window.scrollTo({ top: 0, behavior: 'smooth' });
  });
</script>

Scroll within a container

Track scrolling inside a bounded region with w-scroll-self and update a progress indicator.

<div w-scroll w-scroll-self id="reader"
     style="height: 240px; overflow: auto;">
  <!-- article content -->
</div>
<progress id="bar" max="100" value="0"></progress>

<script>
  const reader = document.getElementById('reader');
  const bar = document.getElementById('bar');
  reader.addEventListener('w-scroll', (e) => {
    const max = reader.scrollHeight - reader.clientHeight;
    bar.value = max > 0 ? (e.detail.scrollTop / max) * 100 : 0;
  });
</script>
