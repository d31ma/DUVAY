Touch

The DuVay equivalent of Vuetify's v-touch directive. Add w-touch to any element to capture swipe gestures and react to direction. DuVay uses native Pointer Events under the hood, so the same handler covers mouse, touch, and pen with one consistent API. Requires duvay-directives.js.

Usage

Mark the gesture surface with w-touch, then listen for the bubbling w-swipe event. On a touch device, try swiping the element in various directions; with a mouse, press and drag.

<div class="w-card" w-touch>Swipe me</div> <script> const surface = document.querySelector('[w-touch]'); surface.addEventListener('w-swipe', (e) => { console.log(e.detail.direction); // 'left' | 'right' | 'up' | 'down' }); </script>

Options

Tune the gesture with a data attribute on the same element.

AttributeDefaultDescription
w-touch-threshold 16 Minimum distance in pixels the pointer must travel before a swipe is registered. Movement shorter than this is treated as a tap and emits no w-swipe.
<div class="w-card" w-touch w-touch-threshold="48"> Needs a longer swipe </div>

Events

Where Vuetify's v-touch takes an object of directional callbacks (left, right, up, down) plus lifecycle callbacks (start, move, end), DuVay emits native bubbling CustomEvents. The four directional callbacks are consolidated into a single w-swipe event that carries the resolved direction in its detail, and the lifecycle callbacks map one-to-one onto w-touchstart / w-touchmove / w-touchend.

EventMirrorsdetail fields
w-swipe left / right / up / down direction ('left' | 'right' | 'up' | 'down'), offsetX, offsetY
w-touchstart start offsetX, offsetY
w-touchmove move offsetX, offsetY
w-touchend end offsetX, offsetY

offsetX and offsetY are the signed distances from the gesture's start point: positive offsetX means rightward, positive offsetY means downward.

Vuetify's four directional callbacks become one w-swipe listener that branches on detail.direction:

const surface = document.querySelector('[w-touch]');

surface.addEventListener('w-swipe', (e) => {
  switch (e.detail.direction) {
    case 'left':  next();      break; // v-touch: { left:  () => next() }
    case 'right': previous();  break; // v-touch: { right: () => previous() }
    case 'up':    collapse();  break; // v-touch: { up:    () => collapse() }
    case 'down':  expand();    break; // v-touch: { down:  () => expand() }
  }
});

// Lifecycle, mirroring v-touch start / move / end:
surface.addEventListener('w-touchstart', (e) => { /* e.detail.offsetX/Y */ });
surface.addEventListener('w-touchmove',  (e) => { /* e.detail.offsetX/Y */ });
surface.addEventListener('w-touchend',   (e) => { /* e.detail.offsetX/Y */ });

Example

Swipe to move between panels of a window. w-swipe advances or rewinds the active index, and the lifecycle events let you follow the pointer for a live drag offset.

<div class="w-window" w-touch> <div class="w-window-item" data-active>Panel 1</div> <div class="w-window-item">Panel 2</div> <div class="w-window-item">Panel 3</div> </div> <script> const win = document.querySelector('.w-window'); const items = [...win.querySelectorAll('.w-window-item')]; let index = 0; function show(i) { index = Math.max(0, Math.min(items.length - 1, i)); items.forEach((el, n) => el.toggleAttribute('data-active', n === index)); } win.addEventListener('w-swipe', (e) => { if (e.detail.direction === 'left') show(index + 1); if (e.detail.direction === 'right') show(index - 1); }); // Optional: track the in-progress drag for a parallax effect. win.addEventListener('w-touchmove', (e) => { win.style.setProperty('--drag-x', e.detail.offsetX + 'px'); }); win.addEventListener('w-touchend', () => { win.style.removeProperty('--drag-x'); }); </script>

Note the compromise: Vuetify exposes four independent directional callbacks, whereas DuVay raises a single w-swipe event and you branch on detail.direction. This keeps the API to one listener and plays naturally with event delegation.
