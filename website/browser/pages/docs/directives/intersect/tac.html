Intersect

The DuVay equivalent of Vuetify's v-intersect directive. The w-intersect attribute wraps the native IntersectionObserver API and provides an easy-to-use interface for detecting when an element becomes visible in the viewport. It fires a bubbling w-intersect event and reflects its state as a w-intersecting attribute, so you can react in JavaScript or style reveals in pure CSS. Requires duvay-directives.js.

Usage

Add w-intersect to any element you want to observe. As it scrolls into and out of view, DuVay updates w-intersecting and dispatches a w-intersect event — for example, a card that flips from error to success the moment it enters the viewport.

<div class="w-card" w-intersect>
  Scroll me into view
</div>

<script>
  document.querySelector('[w-intersect]')
    .addEventListener('w-intersect', (e) => {
      e.target.classList.toggle('is-visible', e.detail.isIntersecting);
    });
</script>

Options

The directive accepts options that map directly onto the Intersection Observer API and Vuetify's v-intersect modifiers. Each is set as a w-intersect-* attribute.

AttributeVue modifierDescription
w-intersect-once.onceStop observing after the first time the element intersects.
w-intersect-quiet.quietSkip the initial callback fired when the observer first attaches.
w-intersect-marginrootMarginGrow or shrink the root bounds, e.g. "0px 0px -100px 0px".
w-intersect-thresholdthresholdComma-separated list of visible fractions that trigger the callback, e.g. "0, 0.5, 1".
w-intersect-rootrootCSS selector for the scroll container to observe within, e.g. "#scroll-area".

An example using the threshold option, which fires only once half of the element is visible:

<div
  class="w-card"
  w-intersect
  w-intersect-threshold="0.5"
>
  Half of me must be visible
</div>

Events

Observed elements dispatch a bubbling w-intersect CustomEvent. The detail object carries isIntersecting (a boolean) and entries (the raw IntersectionObserverEntry[]).

el.addEventListener('w-intersect', (e) => {
  if (e.detail.isIntersecting) {
    console.log('Entered view', e.detail.entries);
  } else {
    console.log('Left view');
  }
});

Reflected state

DuVay reflects the current visibility onto the element as w-intersecting="true|false". Because it is a plain attribute, you can drive reveal animations entirely in CSS — no JavaScript listener required.

<div class="reveal" w-intersect w-intersect-once>
  I fade in when scrolled into view
</div>

<style>
  .reveal {
    opacity: 0;
    transform: translateY(1rem);
    transition: opacity 0.4s ease, transform 0.4s ease;
  }
  .reveal[w-intersecting="true"] {
    opacity: 1;
    transform: none;
  }
</style>

Examples

Reveal on scroll

Combine w-intersect-once with the reflected state attribute to animate elements in exactly once as the page scrolls.

<section
  class="reveal"
  w-intersect
  w-intersect-once
  w-intersect-margin="0px 0px -100px 0px"
>
  <h2>I slide in once</h2>
</section>

Lazy-load images

Defer loading heavy assets until a placeholder approaches the viewport. The negative-free w-intersect-margin pre-loads the image 200px before it appears.

<img
  data-src="/large-image.jpg"
  alt="Deferred image"
  w-intersect
  w-intersect-once
  w-intersect-margin="200px"
>

<script>
  document.querySelectorAll('img[w-intersect]').forEach((img) => {
    img.addEventListener('w-intersect', (e) => {
      if (e.detail.isIntersecting) {
        img.src = img.dataset.src;
      }
    });
  });
</script>

Infinite-scroll sentinel

Place an empty sentinel at the end of a list. Each time it intersects, load the next page of results. Omit w-intersect-once so it keeps firing as the sentinel re-enters view.

<ul id="feed"><!-- items --></ul>
<div id="sentinel" w-intersect w-intersect-margin="0px 0px 400px 0px"></div>

<script>
  document.getElementById('sentinel')
    .addEventListener('w-intersect', (e) => {
      if (e.detail.isIntersecting) loadMore();
    });
</script>

Scoped root

Observe visibility within a scrolling container instead of the viewport by pointing w-intersect-root at it.

<div id="scroll-area" style="overflow: auto; height: 300px">
  <div w-intersect w-intersect-root="#scroll-area">
    Tracked against the panel, not the window
  </div>
</div>
