Mutate

The DuVay equivalent of Vuetify's v-mutate directive. The w-mutate attribute wraps the native MutationObserver API to detect when an element is updated. It requires duvay-directives.js, which wires the observer and dispatches a bubbling w-mutate event whenever the watched node changes.

Usage

Add w-mutate to any element and listen for the w-mutate event. The observer can be configured two ways: through type-flag attributes, or by leaving them off entirely. By default — with no type flag present — w-mutate-attr, w-mutate-char, w-mutate-child, and w-mutate-sub are all enabled. Applying any one type flag manually sets the undefined options to false, exactly matching Vuetify's modifier behavior.

<!-- attr, char, child, and sub are all observed -->
<div w-mutate>...</div>

<!-- child is observed; attr, char, and sub are off -->
<div w-mutate w-mutate-child>...</div>

Combine type flags to scope the observer. The pair below watches child-node additions and removals across the entire subtree:

<div w-mutate w-mutate-child w-mutate-sub id="watched">...</div>

<script>
  document.getElementById('watched').addEventListener('w-mutate', (e) => {
    onMutate(e.detail.mutations);
  });

  function onMutate (mutations) {
    //
  }
</script>

Once

When a callback should fire only once, add w-mutate-once. The observer disconnects itself after the first detected mutation. In the example below the same content is bound to two cards and an input; both cards fire w-mutate on the first change, but the once-flagged card unbinds afterward and ignores every subsequent edit.

<input type="text" id="content-input" value="Foo">

<div class="w-card" id="card-a">Foo</div>
<div class="w-card" id="card-b" w-mutate-once>Foo</div>

<script>
  let mutations = 0;
  const input = document.getElementById('content-input');
  const cardA = document.getElementById('card-a');
  const cardB = document.getElementById('card-b');

  [cardA, cardB].forEach((card) => {
    card.setAttribute('w-mutate', '');
    card.setAttribute('w-mutate-char', '');
    card.addEventListener('w-mutate', () => { mutations++; });
  });

  input.addEventListener('input', () => {
    cardA.textContent = input.value;
    cardB.textContent = input.value; // card-b only counts the first change
  });
</script>

Immediate

Unlike the Intersection Observer API, a Mutation Observer's callback is not invoked when it is created. Add w-mutate-immediate to normalize this: DuVay fires w-mutate once right after wiring, then again on every subsequent mutation. The immediate dispatch carries an empty detail.mutations array, is not counted as a mutation, and does not trigger disconnect when combined with w-mutate-once.

<div w-mutate w-mutate-immediate id="panel">...</div>

<script>
  let mutations = 0;
  document.getElementById('panel').addEventListener('w-mutate', () => {
    mutations++; // fires once immediately, then on each change
  });
</script>

Modifiers

Each DuVay attribute maps directly to a Vuetify v-mutate modifier. With no type flag present, the four observer options default to true; adding any one flag turns the others off.

AttributeVue modifierDefaultDescription
w-mutate-attr.attrtrueMonitors the target node's attribute changes.
w-mutate-char.chartrueMonitors changes to character data (and descendants when w-mutate-sub is set).
w-mutate-child.childtrueMonitors addition and removal of child nodes (and descendants when w-mutate-sub is set).
w-mutate-sub.subtrueExtends monitoring to the entire subtree of the target node.
w-mutate-once.once—Dispatches w-mutate once, then disconnects the observer.
w-mutate-immediate.immediate—Dispatches w-mutate once right after wiring; does not affect w-mutate-once.

Events

The directive dispatches a single bubbling CustomEvent named w-mutate. Its detail.mutations property holds the MutationRecord[] reported by the underlying observer (empty for the immediate dispatch).

el.addEventListener('w-mutate', (e) => {
  e.detail.mutations.forEach((record) => {
    if (record.addedNodes.length) onItemsAdded(record.addedNodes);
  });
});

Example

A live list that reacts whenever rows are inserted or removed. The observer watches the subtree for child-node changes and reports the affected nodes through detail.mutations.

<ul class="live-list" w-mutate w-mutate-child w-mutate-sub> <li>First</li> <li>Second</li> </ul> <script> const list = document.querySelector('.live-list'); list.addEventListener('w-mutate', (e) => { const added = e.detail.mutations .flatMap((record) => [...record.addedNodes]); if (added.length) console.log('rows added:', added.length); }); // appending a row triggers w-mutate const li = document.createElement('li'); li.textContent = 'Third'; list.appendChild(li); </script>
