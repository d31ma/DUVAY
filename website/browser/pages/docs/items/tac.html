Items

Many DuVay components accept a list of choices through an items attribute: selects, autocompletes, breadcrumbs, command palettes, data tables, and more. Items can be a comma-separated string or a JSON array of objects.

String items

The simplest form is a comma-separated list. Each string becomes both the visible label and the value.

Apple
Banana
Cherry

Object items

For separate labels and values, pass a JSON array. By default DuVay reads title for the label and value for the selection value.

Ada Lovelace
Grace Hopper

Custom title and value keys

If your data uses different field names, set item-title and item-value.

<w-autocomplete label="Choose a role" item-title="name" item-value="id" items='[{"name":"Admin","id":"admin"}, {"name":"Editor","id":"editor"}, {"name":"Viewer","id":"viewer"}]'> </w-autocomplete>

Disabled items

Object items can include a disabled property to make an option non-interactive.

Slots instead of items

When you need custom markup per row, omit items and use slotted children. DuVay components that accept items also expose a default slot for hand-authored rows.

<w-autocomplete label="Custom rows"> <button class="w-autocomplete-item" type="button" role="option" value="ada"> <strong>Ada Lovelace</strong> <span class="w-text-subtle">&mdash; First programmer</span> </button> <button class="w-autocomplete-item" type="button" role="option" value="grace"> <strong>Grace Hopper</strong> <span class="w-text-subtle">&mdash; COBOL pioneer</span> </button> </w-autocomplete>

Common item properties

PropertyDescription
title / item-titleThe visible label.
value / item-valueThe submitted or emitted value.
subtitleSecondary text shown beneath the title.
disabledMakes the item non-interactive.
