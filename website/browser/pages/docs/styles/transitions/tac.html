Transitions

DuVay's motion is centralized in three transition tokens. Components reference them for hover, focus, and state changes, keeping animation timing consistent across the framework. Use the same tokens in your own CSS so custom elements match built-in components. All motion is suppressed automatically under prefers-reduced-motion.

Usage

Transition tokens

Reference the tokens directly when writing custom transitions. This keeps timing consistent with the rest of DuVay.

Hover to transition shadow
.my-card { transition: box-shadow var(--w-transition-normal), transform var(--w-transition-fast); } .my-card:hover { box-shadow: var(--w-shadow-md); }

Built-in named transitions

DuVay ships a set of named transition classes for common component animations. These are used internally by components such as expanders, dialogs, and snackbars.

<div class="w-fade-transition">...fade in/out...</div> <div class="w-expand-transition">...height expand...</div> <div class="w-slide-y-transition">...slide vertically...</div>

Reduced motion

Wrap non-essential motion in a prefers-reduced-motion guard, or rely on DuVay's components which already disable transitions and animations when the user requests reduced motion.

@media (prefers-reduced-motion: reduce) { * { transition-duration: 0.01ms !important; } }

Tokens

TokenValueUse
--w-transition-fast120ms easeSmall, immediate feedback (hover tints, icon swaps).
--w-transition-normal140ms easeDefault for most interactive state changes.
--w-transition-slow160ms easeLarger movements such as panels and overlays.
