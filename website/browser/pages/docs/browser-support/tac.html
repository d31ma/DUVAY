Browser support

DuVay targets modern evergreen browsers and the current platform standards. Because the framework is built on CSS custom properties and standard Custom Elements, it works wherever those features are available without polyfills or transpilation.

Supported browsers

DuVay is tested against the latest two stable versions of each major evergreen browser. Older or non-evergreen browsers are not supported.

Graceful degradation

DuVay is built on progressive enhancement, so partial support degrades gracefully rather than breaking:

  • No JavaScript — Components authored with CSS classes are fully styled and usable. Interactive enhancements (dropdowns, dialogs, toasts) simply do not activate.
  • No Custom Elements — Author your markup with CSS classes instead of web-component tags; the visual result is identical.
  • Missing :has() or container queries — Affected components fall back to their default static layout; content remains readable and accessible.

Because the static HTML carries the real DuVay class names, the baseline experience never depends on script execution.

Mobile

DuVay is mobile-first. Touch targets meet the 44x44 px minimum, layouts use logical properties for RTL support, and interactive components are tested on iOS Safari and Chrome for Android.
