Display & Platform

Vuetify exposes a reactive useDisplay composable in JavaScript that reports the current breakpoint, a mobile flag, viewport height / width, and a platform object. DuVay has no JavaScript display API — every one of these concerns is expressed in pure CSS through media queries, breakpoint-prefixed utilities, visibility helpers, and pointer / hover / feature queries. The sections below mirror the Vuetify page topic-for-topic and show the CSS equivalent for each.

Usage

Where Vuetify asks you to read display.mdAndUp in script and toggle behaviour reactively, DuVay lets the browser do it declaratively. Wrap markup in a visibility helper, or apply a breakpoint-prefixed sizing utility, and the layout responds to the viewport with zero script.

API

Vuetify documents a single useDisplay composable. DuVay’s “API” is the CSS surface itself — there is nothing to import or call. The pieces are:

  • Breakpoint media queries — min-width queries at the breakpoint scale.
  • Visibility helpers — w-hide-mobile, w-show-mobile, w-hide-tablet, w-hide-desktop.
  • Responsive sizing — w-{bp}-{size} and h-{bp}-{size} (from that breakpoint up).
  • Platform queries — @media (pointer: coarse), @media (hover: hover), @supports.

Breakpoints and Thresholds

Vuetify defines named thresholds that create “and up” ranges starting at xs (0px). DuVay uses the same Vuetify-parity scale as min-width media queries. Everything below a breakpoint is the implicit base (mobile-first); each query layers on from its threshold upward.

NameDevice classDuVay threshold (min-width)
xsSmall to large phoneBase (0–599px, no prefix)
smSmall to medium tablet600px
mdLarge tablet to laptop960px
lgDesktop1280px
xl4k and ultra-wide1920px

The “and up” mentality is baked in: w-md-50 applies at md and every larger breakpoint, exactly like Vuetify’s mdAndUp.

@media (min-width: 600px)  { /* sm and up */ }
@media (min-width: 960px)  { /* md and up */ }
@media (min-width: 1280px) { /* lg and up */ }
@media (min-width: 1920px) { /* xl and up */ }

Options

Vuetify lets you override thresholds and the mobileBreakpoint at runtime. In DuVay the thresholds are the media-query breakpoints above. To customise them, you change the min-width values in your own CSS rather than passing an options object — the breakpoints are configuration-as-CSS.

/* Your override: a tighter custom scale */
@media (min-width: 540px) { .app-cols { columns: 2; } }
@media (min-width: 800px) { .app-cols { columns: 3; } }

Examples

Vuetify’s example uses a switch on the breakpoint name to set a card height. The CSS equivalent sets the property directly inside each breakpoint query — the same outcome without script.

.responsive-card { height: 220px; }            /* xs base */
@media (min-width: 600px)  { .responsive-card { height: 300px; } }
@media (min-width: 960px)  { .responsive-card { height: 400px; } }
@media (min-width: 1280px) { .responsive-card { height: 500px; } }

Interface

Vuetify’s TypeScript interface exposes name, height, width, the breakpoint booleans, mobile, and a platform object (android, ios, cordova, electron, chrome, edge, firefox, opera, win, mac, linux, touch, ssr). DuVay has no script interface; each field maps to a CSS construct instead:

Vuetify fieldDuVay CSS equivalent
width / height100vw / 100dvh units; no readback needed
name, xs…xlmin-width media queries on the breakpoint scale
mobileBase styles + w-hide-mobile / w-show-mobile
platform.touch@media (pointer: coarse)
platform.ssrHandled by static prerender; not a runtime flag
browser / OS flags@supports feature detection where it matters

Using Setup

Vuetify pairs useDisplay with setup() to, for example, make a dialog fullscreen on mobile. DuVay does the same with a breakpoint rule on the dialog surface — no setup function, no reactive subscription.

/* Fullscreen dialog on phones, centered card from sm up */
.w-dialog { inset: 0; width: 100%; height: 100dvh; border-radius: 0; }
@media (min-width: 600px) {
  .w-dialog { inset: auto; width: 32rem; height: auto; border-radius: var(--w-radius-lg); }
}

Breakpoint conditionals

Vuetify’s breakpoint booleans (smAndUp, mdAndDown, …) follow its grid naming. In DuVay the equivalent conditional is the media query plus, for the common show/hide cases, the visibility helpers. “And up” is min-width; “and down” is max-width.

Component Mobile Breakpoints

Some Vuetify components (v-banner, v-navigation-drawer, v-slide-group) accept a mobile-breakpoint prop that defaults to the global value (lg — mobile when narrower than 1280px). DuVay components carry the same responsive behaviour in their own CSS rather than a prop: the sidebar, for instance, collapses below its breakpoint automatically, and you adjust it by overriding the relevant min-width rule.

Display class overrides

Vuetify offers displayClasses that emit names like v-component--mobile, plus a custom mobileBreakpoint override. DuVay’s parallel is the responsive sizing utility set — w-{bp}-{size} and h-{bp}-{size} — which switch an element’s dimensions at a breakpoint without any JavaScript class toggling.

Platform detection

Vuetify’s platform object reports touch, SSR, browser, and OS at runtime. For input-specific behaviour, prefer CSS capability queries over user-agent sniffing — they describe what the device can actually do, and they cost nothing at runtime.

/* Touch devices: larger hit areas (Vuetify platform.touch) */
@media (pointer: coarse) {
  .w-btn { min-height: 2.75rem; } /* 44px target */
}

/* Devices with a real hover (mouse) */
@media (hover: hover) {
  .w-card:hover { box-shadow: var(--w-shadow-lg); }
}

/* Respect reduced-motion preference */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}

/* Feature detection in place of browser/OS flags */
@supports (backdrop-filter: blur(8px)) {
  .w-dialog-backdrop { backdrop-filter: blur(8px); }
}

SSR is not a runtime flag in DuVay: the docs site prerenders statically, so there is no hydration-mismatch concern to guard against. Everything renders meaningfully without JavaScript and the breakpoints apply on first paint.
