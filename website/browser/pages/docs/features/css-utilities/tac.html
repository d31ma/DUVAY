CSS Utilities

DuVay ships its own compact utility layer — plain classes for spacing, display, flex, text, sizing, borders, and shadows. Utilities are zero-dependency and work without JavaScript, so you can compose layouts directly in your markup alongside DuVay's semantic components. Tailwind and UnoCSS remain optional coexistence paths for teams that already use them.

Native utilities

The utility classes are the primary way to tweak layout and spacing without writing custom CSS. Combine display, flex, gap, and spacing classes to assemble a row in one element.

<div class="d-flex justify-space-between align-center ga-4 pa-4"> <span class="text-h6">Title</span> <button class="w-btn w-btn-filled">Action</button> </div>

Categories

DuVay's utility families mirror the most common styling needs:

  • Spacing — margin and padding on every side (ma-4, pa-4, px-2, pt-6) and grid gaps (ga-4, gr-2, gc-3).
  • Display — d-flex, d-grid, d-block, d-inline-flex, d-none.
  • Flex — flex-column, flex-wrap, justify-space-between, align-center, flex-grow-1.
  • Text — type scale (text-h6, text-body-1), alignment (text-center), weight (font-weight-bold), and emphasis (text-medium-emphasis).
  • Sizing — w-100, h-100, w-50, fill-height.
  • Borders & shadows — rounded, rounded-lg, rounded-pill, elevation-1 through elevation-6.

Every utility resolves to a design token, so values stay consistent with components and respect the active theme. See the full list on the Utilities reference.

Working with other utility systems

DuVay's native utilities cover the common cases, but you can layer a dedicated utility engine on top for highly custom, one-off work. Import duvay.css first, then let the other system generate atomic classes that reference the same CSS custom properties. The four guides below cover each path.

Accessibility

Utilities are presentational. When you use spacing and display classes to build layouts, keep the underlying markup semantic — correct heading order, real buttons and links, and a logical tab order — so the visual arrangement never overrides the document structure.
