Icons

DuVay does not bundle an icon font. Use any icon set you prefer — Material Symbols, Material Design Icons, Font Awesome, Phosphor — by loading its font or SVG set and telling DuVay how to render it. The default adapter renders the icon name as text (ligatures), which matches Material Symbols and keeps everything zero-dependency.

Default ligature rendering

With no configuration, w-icon and component icon attributes render the name as text. This works out of the box with ligature-based fonts such as Material Symbols.

Icon sets

Register an icon set with WIcons.set(), then reference icons by name. DuVay ships adapters for font classes, ligatures, inline SVG paths, and custom elements. This mirrors Vuetify's icons configuration and VIcon resolution, but stays Light-DOM and zero-dependency.

Class-based icons (MDI, Font Awesome)

WIcons.set('mdi', { type: 'class', prefix: 'mdi' });
WIcons.defaultSet = 'mdi';

<w-icon name="home"></w-icon>
<w-btn icon="home" variant="filled">Home</w-btn>
<!-- renders <span class="w-icon mdi mdi-home"></span> -->

Set-scoped icons without changing the default

Use the icon-set attribute to pick an adapter for a single component without affecting the global default.

<w-icon name="search" icon-set="mdi"></w-icon>
<w-btn icon="search" icon-set="mdi" variant="outlined">Search</w-btn>

SVG paths

WIcons.set('w-svg', {
  type: 'svg',
  paths: {
    home: '<path d="M10 20v-6h4v6h5v-8h3L12 3 2 12h3v8z" fill="currentColor"/>'
  }
});

<w-icon name="home" icon-set="w-svg"></w-icon>

Aliases

Define aliases and reference them with a $ prefix.

WIcons.alias('close', 'mdi:windoclose');
WIcons.alias('menu', 'mdi:menu');

<w-icon name="$close"></w-icon>
<w-icon-btn icon="$menu" aria-label="Menu"></w-icon-btn>

Icon buttons

Always provide an aria-label for icon-only controls. See the Buttons page for variants.
