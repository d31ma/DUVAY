Hotkeys

w-hotkey displays a keyboard shortcut as styled <kbd> chips with platform-aware labels — ⌘ on macOS, Ctrl on Windows / Linux. Join simultaneous keys with + and separate sequential steps with -. It is presentational; register the actual listeners yourself (see below).

Basic

Use the keys attribute. By default the platform is detected from the browser.

⌘K

Platform

Force a platform with platform="mac" or platform="pc" — handy for documentation that shows both.

⌘⇧P CtrlShiftP

Display mode

display-mode controls how modifier keys render: icon (default, platform-aware), symbol (always glyphs), or text (always words).

⌃⌥S ControlOptionS

Sequences

Separate sequential steps with - to show a “then” chord, e.g. ctrl+k-ctrl+s.

CtrlKthenCtrlS

Variants & states

Use variant="plain" for unboxed labels, or disabled for an unavailable shortcut. Named keys like enter, esc, tab, space, and the arrows are recognized.

⌘↵ ⌘Z

Binding the shortcut

w-hotkey only renders the keys — wire the behavior with a listener:

document.addEventListener('keydown', (e) => {
  if ((e.metaKey || e.ctrlKey) && e.key.toLowerCase() === 'k') {
    e.preventDefault();
    openSearch();
  }
});

Accessibility

Each w-hotkey carries a readable aria-label (e.g. “Control + K, then Control + S”). Document shortcuts in a help dialog, avoid overriding browser or screen-reader keys, and keep focus visible when a shortcut opens a layer.
