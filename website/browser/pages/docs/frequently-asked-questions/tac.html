Frequently asked questions

Answers to the questions developers ask most often when adopting DuVay. If something is missing here, check the relevant component page or open a discussion on the repository.

Does DuVay have any dependencies?

No. DuVay is a zero-dependency framework. The stylesheet is plain CSS built on custom properties, and the optional behavior layer and web components are vanilla JavaScript. There is no runtime, no virtual DOM, and nothing to install beyond the files themselves.

Do I need a build step?

No. You can link the hosted CSS and scripts directly and start writing markup. There is no Sass, PostCSS, or bundler required for the framework itself. A build step is only relevant if you choose to self-host and minify your own copies.

How does theming work?

Themes are driven entirely by the data-w-theme attribute on the root element. Every color, radius, shadow, and spacing value resolves from a CSS custom property, so switching themes is a single attribute change.

<html data-w-theme="light"> <!-- light theme --> <html data-w-theme="dark"> <!-- dark theme --> <html data-w-theme="auto"> <!-- follows prefers-color-scheme -->

The auto theme respects the operating system prefers-color-scheme setting and updates live. See the Themes page for the full token set.

What is the difference between the CSS classes and the web components?

DuVay ships two ways to use every component, and they share the exact same styles:

ApproachLooks likeBest for
CSS classes<button class="w-btn w-btn-filled">Static markup, server-rendered pages, full control of HTML
Web components<w-btn variant="filled">Less markup, declarative attributes, encapsulated behavior

The web components use Light DOM only (no Shadow DOM), so they render the same DuVay class names you would write by hand. You can mix both approaches freely on the same page.

Does DuVay work without JavaScript?

Yes, for everything that is structurally static. DuVay is built on progressive enhancement: a button, card, alert, or list rendered with CSS classes is fully styled and usable with no JavaScript at all. The behavior layer and web components enhance interactivity (dropdowns, dialogs, toasts, theme cycling) once the scripts load.

Is DuVay tied to a framework like React or Vue?

No. DuVay is framework-agnostic. The CSS works anywhere you can write HTML, and the web components are standard Custom Elements, so they work in plain HTML, React, Vue, Svelte, Angular, or any templating language. There is nothing React- or Vue-specific in the framework.

Is DuVay accessible?

Accessibility is a core requirement, not an afterthought. Components ship with correct role and aria-* attributes, full keyboard navigation, logical focus order, visible :focus-visible indicators, and respect for prefers-reduced-motion. Color tokens meet WCAG AA contrast ratios, and a high-contrast theme is available.

Which browsers are supported?

DuVay targets modern evergreen browsers (Chrome, Edge, Firefox, and Safari). See the Browser support page for the full matrix and the underlying platform features.

How are versions numbered?

DuVay uses CalVer (YY.WW.DD). Versioned CDN URLs are permanent, so you can pin a release and trust it never changes underneath you.
