Density and sizing

DuVay uses two levers to control how compact or prominent a component feels: density adjusts internal padding and height, while size scales the whole control. Both work on CSS classes and web components.

Density

Density is applied through the density attribute or the matching w-density-* utility class. Not every component implements every density.

ValueClassEffect
compactw-density-compactReduces height and vertical padding (buttons, inputs, chips, list items).
comfortablew-density-comfortableDefault-ish spacing with a 44 px minimum touch target.
prominentw-density-prominentEnlarges app bars and toolbars (typically 72 px).
Compact Comfortable Default

Sizing

Components that support a size attribute accept a t-shirt scale. Buttons map them to w-btn--xs, w-btn--sm, w-btn--lg, and w-btn--xl. Other components (icons, avatars, inputs) may use the same tokens with their own class names.

Attribute valueButton classTypical use
xs / x-smallw-btn--xsDense toolbars, chips, icon-only actions.
sm / smallw-btn--smSecondary actions, compact forms.
default (omit)w-btn--mdPrimary actions, default forms.
lg / largew-btn--lgCalls to action, empty states.
xl / x-largew-btn--xlHero actions, marketing surfaces.
Extra small Small Default Large Extra large

Combining density and size

Density and size are independent. A button can be large and compact, or small and comfortable. Use the combination that fits your layout and touch-target requirements.

Large compact Small comfortable

CSS utility reference

If you are styling plain HTML, use the utility classes directly. The web-component layer simply maps attributes to these classes.

<!-- Density --> <button class="w-btn w-btn--filled w-density-compact">Compact</button> <!-- Size --> <button class="w-btn w-btn--filled w-btn--lg">Large</button> <!-- Both --> <button class="w-btn w-btn--filled w-btn--lg w-density-compact">Large compact</button>
