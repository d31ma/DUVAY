Item Groups

Item groups coordinate selected state across arbitrary children. They support single or multiple selection, a selection cap, custom active classes, and full keyboard navigation.

Multiple selection with max

Add multiple to allow more than one selection and max to cap how many items can be selected at once.

Custom active class

Use active-class (or selected-class) to apply your own CSS class to selected items.

Keyboard navigation

Item groups implement roving tabindex and arrow-key navigation. Tab into the group, then use ← → ↑ ↓ to move focus, Home and End to jump to the first or last item, and Space or Enter to toggle the focused item.

API reference

Relevant web component: <w-item-group>.

Attribute Type Default Description
value string "" Selected value(s). For multiple, use a JSON array like ["a","b"].
multiple boolean false Allow more than one item to be selected.
mandatory boolean false At least one item must always be selected.
max number unlimited Maximum number of selectable items when multiple is set.
active-class / selected-class string "active" CSS class applied to selected items. active-class takes precedence.
disabled boolean false Disable the entire group.

Events

  • change — fired when the selection changes. The event detail has a value property. In multiple mode, that property is an array.
